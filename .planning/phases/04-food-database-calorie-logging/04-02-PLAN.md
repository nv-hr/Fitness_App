Create backend/src/repositories/food.repository.js following the established repository pattern:

1. Import: `import { pool } from '../config/database.js'` and `import { AppError } from '../utils/errors.js'`

2. Export `searchFoods(userId, query, limit = 20)`:
   - Query: `SELECT id, name, calories_per_100g, category FROM foods WHERE (is_custom = FALSE OR user_id = ?) AND name LIKE ? ORDER BY CASE WHEN is_custom = FALSE THEN 0 ELSE 1 END, name LIMIT ?`
   - Parameters: `[userId, '%' + query + '%', limit]`
   - Returns: array of food objects
   - Wrap errors with AppError('DatabaseError', ...)

3. Export `createCustomFood(userId, { name, calories_per_100g, category })`:
   - Query: `INSERT INTO foods (user_id, name, calories_per_100g, category, is_custom) VALUES (?, ?, ?, ?, TRUE)`
   - Return: `SELECT * FROM foods WHERE id = LAST_INSERT_ID()`
   - Handle ER_DUP_ENTRY → AppError 409

4. Export `createFoodLog(userId, { foodId, customFoodName, calories, portionGrams, logDate, mealType })`:
   - Query: `INSERT INTO food_logs (user_id, food_id, custom_food_name, calories, portion_grams, log_date, meal_type) VALUES (?, ?, ?, ?, ?, ?, ?)`
   - Return: `SELECT * FROM food_logs WHERE id = LAST_INSERT_ID()`

5. Export `getDailyLogs(userId, logDate)`:
   - Query: `SELECT fl.*, f.name as food_name, f.calories_per_100g FROM food_logs fl LEFT JOIN foods f ON fl.food_id = f.id WHERE fl.user_id = ? AND fl.log_date = ? ORDER BY fl.meal_type, fl.created_at`
   - Returns: array of log entries with food names
   - Also export `getDailyTotal(userId, logDate)`: `SELECT SUM(calories) as total FROM food_logs WHERE user_id = ? AND log_date = ?`

6. Export `getLogHistory(userId, days = 7)`:
   - Query: `SELECT log_date, SUM(calories) as total_calories, COUNT(*) as entry_count FROM food_logs WHERE user_id = ? AND log_date >= DATE_SUB(CURDATE(), INTERVAL ? DAY) GROUP BY log_date ORDER BY log_date DESC`
   - Returns: array of daily summaries

7. Export `getRecentFoods(userId, limit = 10)`:
   - Query: `SELECT DISTINCT COALESCE(f.name, fl.custom_food_name) as name, fl.food_id, fl.calories, fl.portion_grams FROM food_logs fl LEFT JOIN foods f ON fl.food_id = f.id WHERE fl.user_id = ? ORDER BY fl.log_date DESC, fl.created_at DESC LIMIT ?`
   - Returns: array of recently logged foods for quick-add (LOG-05)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → /api/food/search | User-controlled search query, potential SQL injection |
| Client → /api/food (POST) | User-controlled food data, potential XSS via food names |
| Client → /api/food/log (POST) | User-controlled log entries, portion/calorie manipulation |
| /api/food/summary → profile data | Cross-resource data access (food logs + profile) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Injection | searchFoods repository | mitigate | Parameterized LIKE query with `?` placeholder, no string concatenation in SQL |
| T-04-05 | XSS | Custom food name display | mitigate | Frontend must escape food names (React auto-escapes by default); backend validates name length 1-100 |
| T-04-06 | Tampering | Food log calorie values | mitigate | For seeded foods, calculate calories server-side from calories_per_100g × portion_grams; ignore client-supplied calories |
| T-04-07 | Authorization | User-scoped queries | mitigate | All queries use req.user.userId from JWT; no query accepts userId from client |
| T-04-08 | DoS | Search-as-you-type endpoint | mitigate | Rate limiter at 200 req/15min; minimum 2-char search query; LIMIT 20 on results |
</threat_model>
