```
POST /api/auth/register  → 201 { success: true, data: { user: { id, email, pdp_consent } } }
  Body: { email: string, password: string (min 8 chars), pdpConsent: boolean }
  Sets httpOnly cookie: token

POST /api/auth/login  → 200 { success: true, data: { user: { id, email, pdp_consent } } }
  Body: { email: string, password: string }
  Sets httpOnly cookie: token
  Errors: 401 { success: false, error: { message, code: 'AUTHENTICATION_ERROR' } }

POST /api/auth/logout  → 200 { success: true, data: { message: 'Logged out successfully' } }
  Clears httpOnly cookie: token

GET /api/auth/me  → 200 { success: true, data: { id, email, pdp_consent, created_at } }
  Requires: httpOnly cookie 'token' with valid JWT
  Errors: 401 { success: false, error: { message, code: 'AUTHENTICATION_ERROR' } }

GET /api/auth/google  → 302 redirect to Google OAuth consent screen
GET /api/auth/google/callback  → 302 redirect to frontend with httpOnly cookie set
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Backend API via httpOnly cookie | Cookie sent automatically with credentials: 'include' — CSRF surface |
| User input → React form → API call | XSS via form input if not sanitized (React auto-escapes by default) |
| Google OAuth redirect → Frontend | User redirected from Google callback — no token in URL (cookie-only) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-13 | Tampering | Form input → API request | mitigate | Zod schema validation on both client (react-hook-form) and server (express-validator). React auto-escapes JSX content preventing XSS |
| T-01-14 | Spoofing | Google OAuth link | mitigate | Direct <a href="/api/auth/google"> link — no JavaScript manipulation of OAuth URL. Backend Passport validates Google response server-side |
| T-01-15 | Information Disclosure | ProtectedRoute client-side guard | mitigate | ProtectedRoute is UX convenience only — real auth enforcement is server-side via authenticateToken middleware. If cookie is missing, API returns 401 regardless of frontend routing |
| T-01-16 | CSRF | httpOnly cookie with credentials: include | mitigate | sameSite: 'lax' on backend cookie prevents cross-site POST requests. GET /api/auth/google is safe (idempotent OAuth redirect). POST endpoints (register, login, logout) are same-site only |
</threat_model>
