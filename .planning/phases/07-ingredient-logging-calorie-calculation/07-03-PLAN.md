```javascript
export async function getRecentFoods(userId, limit = 10) {
  try {
    const [rows] = await pool.query(
      `SELECT COALESCE(f.name, fl.custom_food_name) as name,
              fl.food_id,
              fl.calories,
              MAX(fl.portion_grams) as last_portion_grams
       FROM food_logs fl
       LEFT JOIN foods f ON fl.food_id = f.id
       WHERE fl.user_id = ?
       GROUP BY COALESCE(f.name, fl.custom_food_name), fl.food_id, fl.calories
       ORDER BY MAX(fl.log_date) DESC, MAX(fl.created_at) DESC
       LIMIT ?`,
      [userId, limit]
    );
    return rows;
  } catch (err) {
    throw new AppError('DatabaseError', `Failed to get recent foods: ${err.message}`, 500);
  }
}
```

Key changes:
- `MAX(fl.portion_grams) as last_portion_grams` — gets the largest portion logged for each food
- `GROUP BY COALESCE(f.name, fl.custom_food_name), fl.food_id, fl.calories` — groups by food identity
- `ORDER BY MAX(fl.log_date) DESC, MAX(fl.created_at) DESC` — orders by most recent log date

2. No changes needed in food.controller.js getRecentFoods (line 167-174) — it already returns the repository result directly. The new `last_portion_grams` field will be included automatically.
